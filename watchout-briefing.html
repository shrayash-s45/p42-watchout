Watchout Investors — what the API returns, and its limitations

A plain-English record of the responses the Watchout Investors (PRIME) regulatory-defaulter API returned during live testing — what comes back, in what form, and the limitations observed.

Captured from real API responses, June 2026 · sandbox/test credentials · 17 queries logged below.

01Key findings

When a query lands, the content is rich and specific — regulator, order date, charges, action, often the ₹ amount, the source-order PDF, and a dated case-history trail. The limitations are in how it can be queried and what it will return:

  • It cannot be searched by CIN or DIN. Identifier lookups return “no record” even for entities that are in the registry by name.
  • Search is by name only — and a name search returns many different people/firms, with no way to confirm which one is your subject.
  • Large / high-profile defaulters frequently return A500 — more than 250 matches, and the API then returns nothing. There is no pagination.
  • Identifier coverage inside records is inconsistent — some entities’ records carry CIN+PAN, others carry none.
  • Financial amounts are embedded in free text, not structured fields.
A100 = records returned A200 = no record found A500 = >250 matches, nothing returned A210 = query rejected

04The limitations — with evidence

① A name search returns many names — and you cannot verify which one is your subject primary limitation

Because lookups are by name and there is no working identifier filter, a search returns every defaulter whose name contains your text — frequently many different people or firms. Most records carry no CIN / DIN / PAN, so there is no reliable way to confirm which match (if any) is the entity you are investigating.

Live example — searching the director “Indu Bhushan” returned A100 · 7 records, but they are small personal loan-default notices (₹4.5L–₹75L, “along with other entities/persons”) belonging to different individuals — none is the director we were checking, and not one record carried a DIN to settle it.

A common name compounds this: the more common the name, the more unrelated matches you get, and the less able you are to tell them apart. A name match is a lead, not a confirmed identification.

② You cannot search by CIN or DIN structural gap

Looking up an exact company/director by its registered number does not work — identifier searches return A200 (no record) even for entities that are in the registry by name.

⑤ Financial amounts are buried in free text parsing needed

The ₹ figures are not structured fields — they sit inside the action sentence, e.g. “SUIT FILED- RS.48,21,31,894; BANK OF MAHARASHTRA” or “IMPOSED PENALTY RS.35,00,000”. Extracting a clean number requires parsing, and the wording varies. Dates are formatted DD-Mon-YYYY.

⑥ Operational constraints access & reliability

  • IP whitelisting: the API and its source-document PDFs respond only to pre-registered IP addresses; from any other network the service is unreachable.
  • Rate limits: 6 calls/minute · 350/day (peak) · 50,000/month.
  • Repeat-query flakiness: an identical query that returned records once occasionally came back empty on an immediate re-run.
